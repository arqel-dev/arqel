# v0.10.0 quality foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Three coordinated fixes (release pipeline matrix, recursive form schema serialization, panel-prefix-aware Edit/Create pages) that close out the v0.9.x hotfix chain and unlock the create/edit forms end-to-end before opening Fase 2 of the roadmap.

**Architecture:** PHP-only fixes plus YAML edit. The release pipeline matrix converts to `include` objects so the meta-package `framework` (sourced from `packages/arqel/`) can be split alongside its 19 sub-packages. `Form::toArray()` gains a private recursive `serializeSchema()` helper so layout components emit `entry.schema` populated with their children. `InertiaDataBuilder::resourceMeta()` adds a `panelPath` field; `ArqelEditPage` and `ArqelCreatePage` use it instead of relative URLs. All three fix go in a single PR for coherent semantic + single CI run.

**Tech Stack:** PHP 8.3+, Laravel 12+, Pest 3, Composer 2, Packagist, NPM, GitHub Actions, splitsh-lite, TypeScript 5.6+, Inertia 3.

**Spec:** [`docs/superpowers/specs/2026-05-07-v0.10.0-quality-foundation-design.md`](../specs/2026-05-07-v0.10.0-quality-foundation-design.md)

**Constraint operacional (resource economy):** scope-limited Pest commands (`vendor/bin/pest <path>`); never `pest` global; never `pnpm test` global; subagent dispatches consolidated; reuse `arqel-test` sandbox via in-place upgrade.

```php
/**
 * Serialise the form for the Inertia payload.
 *
 * @return array<string, mixed>
 */
public function toArray(): array
{
    return [
        'schema' => $this->serializeSchema($this->schema),
        'columns' => $this->columns,
        'model' => $this->model,
        'inline' => $this->inline,
        'disabled' => $this->disabled,
    ];
}

/**
 * Recursively serialise a schema array (top-level or component
 * children). Components emit `entry.schema` populated with their
 * children; `Component::toArray()` itself remains "without
 * descending the schema" — Form descends, Component only emits
 * its own props.
 *
 * @param array<int, mixed> $items
 * @return array<int, array<string, mixed>>
 */
private function serializeSchema(array $items): array
{
    $entries = [];
    foreach ($items as $item) {
        if ($item instanceof Field) {
            $entries[] = [
                'kind' => 'field',
                'name' => $item->getName(),
                'type' => $item->getType(),
            ];
        } elseif ($item instanceof Component) {
            $entries[] = [
                'kind' => 'layout',
                ...$item->toArray(),
                'schema' => $this->serializeSchema($item->getSchema()),
            ];
        }
    }
    return $entries;
}
```

(The `'schema' => ...` line **after** the `...$item->toArray()` spread overwrites any `schema` that `Component::toArray()` might emit. Defensive: currently it emits none, but keeping the order ensures the recursive descent always wins.)

Imports already present (`Field`, `Component`); no new imports needed.
