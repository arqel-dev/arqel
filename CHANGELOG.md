# Changelog

Todas as alterações notáveis a este projeto são documentadas neste arquivo.

O formato é baseado em [Keep a Changelog](https://keepachangelog.com/pt-BR/1.1.0/),
e este projeto adere ao [Semantic Versioning](https://semver.org/lang/pt-BR/).

## [Unreleased]

### Added

- Estrutura inicial do monorepo (INFRA-001)
- Orquestração pnpm workspace + Composer path repositories (INFRA-002)
- Toolchain de lint/format (Pint, PHPStan, Biome) + git hooks (Husky + lint-staged + commitlint) (INFRA-003)
- Pipeline CI/CD em GitHub Actions (ci, test-matrix, security, docs-deploy, release) + Dependabot grouping (INFRA-004)
- Renovate Bot config com grouping completo + Dependabot reduzido a github-actions (INFRA-005)
- Política de segurança expandida (SLAs, disclosure process, âmbito) (GOV-001)
- Guia de contribuição + PR template + Issue templates (bug/feature/question) (GOV-003)
- Esqueleto do pacote `arqel/core` com composer.json, PSR-4, README, SKILL, phpunit/pest configs (CORE-001)
- `Arqel\Core\ArqelServiceProvider` com auto-discovery, registos singleton (`ResourceRegistry`, `PanelRegistry`), facade `Arqel`, `config/arqel.php`, comando `arqel:install` e suite Pest com Orchestra Testbench (CORE-002)
- Comando `arqel:install` com Laravel Prompts: publica `config/arqel.php`, faz scaffold de `app/Arqel/Resources`, `app/Arqel/Widgets`, `resources/js/Pages/Arqel`, gera `App\Providers\ArqelServiceProvider`, `resources/views/arqel/layout.blade.php` e `AGENTS.md` (RF-DX-08). Stubs em `packages/core/stubs/`. Flag `--force` sobrescreve sem prompt (CORE-003)
- Contract `Arqel\Core\Contracts\HasResource` (7 métodos estáticos: model, slug, label, plural, navigation icon/group/sort) e `Arqel\Core\Resources\ResourceRegistry` com `register`, `registerMany`, `discover` (PSR-4 + Symfony Finder), `findByModel`, `findBySlug`, `has`, `clear` e `all`. Idempotente; `InvalidArgumentException` em classes que não implementem o contract (CORE-004)
- `Arqel\Core\Panel\Panel` fluent builder (path, brand, theme, primaryColor, darkMode, middleware, resources, widgets, navigationGroups, authGuard, tenant) com getters tipados; `Arqel\Core\Panel\PanelRegistry` create-or-get com `panel(id)`, `setCurrent`/`getCurrent`, `all`, `has`, `clear`; `PanelNotFoundException` em `setCurrent` desconhecido. Routing real adiado para CORE-006 (`ResourceController`) (CORE-005)
- Classe abstract `Arqel\Core\Resources\Resource` (base para Resources do utilizador): static props `$model/$label/$pluralLabel/$slug/$navigationIcon/$navigationGroup/$navigationSort/$recordTitleAttribute`, auto-derivation de slug (`UserResource`→`users`)/label/plural label, lifecycle hooks (`beforeCreate`/`afterCreate`/`beforeUpdate`/`afterUpdate`/`beforeSave`/`afterSave`/`beforeDelete`/`afterDelete`), `recordTitle`/`recordSubtitle`/`indexQuery`. `LogicException` quando `$model` não é declarado. Implementa `HasResource`, `HasFields`, `HasActions`. Contracts adicionados: `HasFields` (requer `fields(): array`), `HasActions` (marker), `HasPolicies` (`getPolicy()`) (CORE-008)
- Comando `arqel:resource {model} {--with-policy} {--force}`: resolve `App\Models\X` para nomes curtos, gera `app/Arqel/Resources/{Model}Resource.php` a partir de `stubs/resource.stub`, respeita `arqel.resources.path/namespace`, `make:policy` interno em `--with-policy`. `--from-model`/`--from-migration` adiados até `Field` existir (FIELDS-*) (CORE-009)
- Blade root view `arqel::app` (`resources/views/app.blade.php`): doctype, `<title inertia>`, CSRF token, FOUC guard de tema (system/light/dark via `localStorage`), `@routes` (Ziggy opcional), `@viteReactRefresh`, `@vite`, `@inertiaHead`, `@inertia`. Config `arqel.inertia.root_view` aponta para `arqel::app`. Publicável via `vendor:publish --tag=arqel-views` (CORE-012)
- Traduções em `en` e `pt_BR` (`resources/lang/{locale}/{messages,actions,table,form,validation}.php`) com namespace `arqel::*`. Cobre actions, confirmation, flash, errors, table sort/pagination/filtros/bulk, form submit/reset/required, validation failed (CORE-013)
- SKILL.md actualizado para reflectir o âmbito real entregue (sem prometer `ResourceController`, `FieldSchemaSerializer`, `Renderable` que ainda não existem)
- Esqueleto do pacote `arqel/fields` (PHP 8.3+, Laravel 12|13, depende de `arqel/core` via path repository): `FieldServiceProvider` registado via auto-discovery, PSR-4 `Arqel\Fields\` → `src/`, Pest + Orchestra Testbench, smoke tests verificando boot do provider e autoload do namespace. Adicionado a `require-dev` do monorepo root como path repo. Tipos concretos e `FieldFactory` chegam em FIELDS-002+ (FIELDS-001)
- `Arqel\Fields\Field` abstract base com construtor `final` (subclasses só declaram `$type`/`$component`), auto-derivation de label a partir do nome (`first_name`→"First Name"), fluent API (`label`/`placeholder`/`helperText`/`default`/`readonly`/`disabled`/`columnSpan`/`columnSpanFull`/`dehydrated`/`live`/`liveDebounced`/`afterStateUpdated`), getters tipados, `isDisabled`/`isDehydrated` aceitam `Closure(?Model)` para avaliação contextual, `getTypeSpecificProps()` por defeito retorna `[]`. Concerns (`HasValidation`/`HasVisibility`/`HasDependencies`/`HasAuthorization`) não estão aplicados ainda — chegam em FIELDS-015..018 (FIELDS-002)
- `Arqel\Fields\FieldFactory` com infrastructure transversal (sem factory methods concretos, que chegam com cada tipo): `register(type, fieldClass)` valida via `is_subclass_of`, `hasType`, `macro(name, Closure)`, `hasMacro`, `flush` (apenas para testes), `__callStatic` que prefere macros sobre registry e lança `BadMethodCallException` para chamadas desconhecidas. Resolve dilema de nomenclatura (não colide com `Field` abstract); alias público `Field::*` virá quando os tipos concretos chegarem (FIELDS-003)
- 6 tipos de input texto em `src/Types/`: `TextField` (base extensível com `maxLength`/`minLength`/`pattern`/`autocomplete`/`mask`), `TextareaField` (rows/cols), `EmailField` (default rule `email`), `UrlField` (default rule `url`), `PasswordField` (`revealable`), `SlugField` (`fromField`/`separator` por defeito `-`). Registados no `FieldServiceProvider::packageBooted` como `text`/`textarea`/`email`/`url`/`password`/`slug`. `getTypeSpecificProps()` filtra valores `null` para payload limpo. Default rules expostas via `getDefaultRules()` aguardando integração com `HasValidation` (FIELDS-015) (FIELDS-004)
- `NumberField` (extensível, `min`/`max`/`step`/`integer()`/`decimals(int)`) com `getDefaultRules()` que emite `numeric`/`integer` + `min:`/`max:` conforme configurado. `CurrencyField` extends Number com `prefix` (default `$`), `suffix` opcional, `thousandsSeparator` (default `,`), `decimalSeparator` (default `.`), `decimals` default 2. Registados como `number`/`currency`. PT-BR usa `prefix('R$')->thousandsSeparator('.')->decimalSeparator(',')` (FIELDS-005)
- `BooleanField` (extensível, `type='boolean'`, `component='Checkbox'`, default `false`, `inline()` para layout horizontal vs stacked) com `getDefaultRules() = ['boolean']`. `ToggleField` extends Boolean (`type='toggle'`, `component='Toggle'`) com `onColor`/`offColor`/`onIcon`/`offIcon` opcionais filtrados quando não definidos. Registados como `boolean`/`toggle` (FIELDS-006)
- `SelectField` (extensível) com 3 modos de options (estáticas via `options(array)`, lazy via `options(Closure)`, relationship via `optionsRelationship(relation, display, ?query)` — resolução adiada para controller em CORE-006). Setters `searchable`/`multiple`/`native`/`creatable`/`createOptionUsing`/`allowCustomValues`. `MultiSelectField` (final, multiple=true, native=false). `RadioField` (final, native=false). Registados como `select`/`multiSelect`/`radio` (FIELDS-007)
- `BelongsToField` (final) configurada via factory `make($name, $relatedResource)` (Field::__construct é final por design FIELDS-002). Valida `is_subclass_of(..., HasResource::class)` em runtime, deriva `relationshipName` removendo sufixo `_id`. Setters `searchable`/`preload`/`searchColumns`/`optionLabel(Closure)`/`relationship(name, ?query)`. `HasManyField` (final, readonly Phase 1) com `canAdd`/`canEdit` aceites como flags forward-compat (Phase 2). Search/preload/create routes são metadata; resolução real adiada para CORE-006. Registados como `belongsTo`/`hasMany` (FIELDS-008)
- `DateField` (extensível, default `format='Y-m-d'`/`displayFormat='d/m/Y'`) com `minDate`/`maxDate` aceitando `string|Closure` (closures resolvidas em serialize-time, retornos não-string descartados graciosamente), `format`/`displayFormat`/`closeOnDateSelection`/`timezone`. `getDefaultRules()=['date']`. `DateTimeField` (final extends Date) com `format='Y-m-d H:i:s'`/`displayFormat='d/m/Y H:i'` e `seconds(bool)` que flipa display para `H:i:s`. Registados como `date`/`dateTime` (FIELDS-009)
- `FileField` (extensível, config-only) com `disk`/`directory`/`visibility`/`maxSize(int kilobytes)`/`acceptedFileTypes(array mimes)`/`multiple`/`reorderable` (auto-flips multiple)/`using(strategy)`. Constantes `STRATEGY_DIRECT`/`STRATEGY_SPATIE_MEDIA_LIBRARY`/`STRATEGY_PRESIGNED` e `VISIBILITY_PRIVATE`/`VISIBILITY_PUBLIC`. `getDefaultRules()` emite `file|array` + `max:` + `mimetypes:` para single-file mode. `ImageField` (final extends File) com defaults de mime gate (`image/jpeg`/`png`/`webp`), `imageCropAspectRatio`/`imageResizeTargetWidth`. `handleUpload`/`handleDelete` e endpoint POST adiados para CORE-006. Registados como `file`/`image` (FIELDS-010)
- `ColorField` (final) com `presets(array)`/`format('hex'|'rgb'|'hsl')`/`alpha(bool)` e constantes tipadas `FORMAT_HEX/RGB/HSL`. `HiddenField` (final, type='hidden', component='HiddenInput') para passing IDs sem UI. `SlugField` extendido com `reservedSlugs(array)` (emite rule `not_in:`) e `unique(class-string, ?column)` (emite rule `unique:table,col` resolvendo `getTable()` quando disponível, com fallback heurístico). Registados como `color`/`hidden` (`slug` já em FIELDS-004) (FIELDS-011)
- `Arqel\Fields\ValidationBridge` traduz arrays de regras Laravel para uma string de schema Zod (ex: `z.string().email().min(1).max(255).nullable()`). 19 translators built-in: `string`/`numeric`/`integer`/`boolean`/`array`/`date`/`file`/`image`/`email`/`url`/`uuid`/`min`/`max`/`size`/`regex`/`in`/`not_in`/`unique`/`nullable`/`required`/`mimetypes`. `register(string, Closure)` para custom rules; `hasRule()`, `flush()` (testes). Unknown rules saltadas silenciosamente para não quebrar com regras server-only (`confirmed`). Helper `Translation` agrega tipo + chain calls + flags `nullable`/`required` (FIELDS-012)
- 21 snapshot tests do shape JSON canónico de cada Field type (`tests/Snapshots/{type}.json`) — funcionam como contrato visível para o lado React e detectam mudanças acidentais de payload. Self-bootstrap (cria snapshot na primeira run, valida byte-equality nas seguintes). Feature tests de search/upload/createOption adiados para CORE-006 (controller) (FIELDS-013 parcial)
- `packages/fields/SKILL.md` reescrito para reflectir o estado real (21 tipos entregues + ValidationBridge + snapshots) com tabela completa de tipos, exemplos copy-pasteáveis (Resource típico, Currency PT-BR, custom select), guia "Creating custom fields" passo-a-passo, secção Macros, ValidationBridge, conventions e anti-patterns (FIELDS-014)
- `Arqel\Fields\Concerns\HasValidation` trait aplicada à classe base `Field`: `required(bool|Closure)` (idempotente, `false` faz drop), `nullable`, `rules(array)`, `rule(string|object, ?$message)`, `unique(?$table, ?$column, mixed $ignorable)`, `maxLength`/`minLength`, `requiredIf`/`requiredWith`/`requiredWithout`, `validationAttribute`, `validationMessage`. `getValidationRules()` resolve Closures e dedupe contra `getDefaultRules()` (subclasses). Rule objects (e.g. `new Password(8)`) preservados verbatim. `SlugField::unique` renomeado para `uniqueIn` para não colidir com a nova API base. Snapshots regenerados para incluir `rules` array (FIELDS-015)
- `Arqel\Fields\Concerns\HasVisibility` trait aplicada à classe base `Field`: 4 contextos (`create`/`edit`/`detail`/`table`) controlados via `hidden(bool)`, `hiddenOnCreate`/`hiddenOnEdit`/`hiddenOnDetail`/`hiddenOnTable`, `visibleOn(string|array)` (whitelist), `hiddenOn(string|array)`, `visibleIf(Closure)` / `hiddenIf(Closure)` (mutuamente exclusivos — `LogicException` se combinados). `isVisibleIn(context, ?Model)` é o oracle único. `InvalidArgumentException` em context names desconhecidos (FIELDS-016)
- `Arqel\Fields\Concerns\HasDependencies` trait aplicada à classe base `Field`: `dependsOn(string|array)` com dedupe automático, `resolveOptionsUsing(Closure)`, `getDependencies`/`hasDependencies`/`getResolveOptionsCallback`. `handleDependencyUpdate(formState, changedField)` é o oracle server-side (controller CORE-006 vai chamá-lo): retorna `[]` quando não há resolver ou quando o changedField não está nas deps; resolver retorno não-array é coerced para `[]`. Quando não há deps declaradas, qualquer changedField dispara o resolver (manual refresh) (FIELDS-017)
- `Arqel\Fields\Concerns\HasAuthorization` trait aplicada à classe base `Field`: `canSee(Closure)` e `canEdit(Closure)` registam predicados; `canBeSeenBy(?Authenticatable, ?Model)` e `canBeEditedBy(?Authenticatable, ?Model)` são oracles default-true. `canBeEditedBy` cascata `canBeSeenBy=false` para `false`. `HasManyField::canAdd`/`canEdit` renomeados para `canAddRecords`/`canEditRecords` para não colidir (FIELDS-018)
- `Arqel\Fields\EagerLoadingResolver::resolve(array $fields): array<int, string>` extrai relation names de `BelongsToField` e `HasManyField` para `Builder::with(...)`. Dedupe automático, ignora non-Field entries graciosamente, honra `relationship()` overrides. Integração no `ResourceController::index()` virá com CORE-006 (RF-P-03, ADR-003 §Consequences) (FIELDS-019)
- `Arqel\Fields\ArqelFields::skill()` retorna inventário JSON-serialisable dos tipos registados + macros (consumido pelo MCP server na Fase 2). `FieldFactory::getRegisteredTypes()` e `FieldFactory::getRegisteredMacros()` expostos como API pública. Comando Artisan `arqel:field {name} {--force}` gera `app/Arqel/Fields/{Name}Field.php` (PHP) + `resources/js/Arqel/Fields/{Name}Input.tsx` (React stub) + imprime hint de registo no ServiceProvider. Stubs em `packages/fields/stubs/field.stub` + `field-component.stub` (FIELDS-022)
- Esqueleto do pacote `arqel/table` (PHP 8.3+, Laravel 12|13, depende de `arqel/core`): `TableServiceProvider` via auto-discovery, smoke tests + skeleton (TABLE-001)
- `Arqel\Table\Table` fluent builder com columns/filters/actions/bulkActions/toolbarActions, `defaultSort`, `perPage`, `searchable/selectable/striped/compact`, `emptyState`. `toArray()` serializa schema para Inertia payload (TABLE-002)
- `Arqel\Table\Column` abstract + 9 tipos concretos (`TextColumn`/`BadgeColumn`/`BooleanColumn`/`DateColumn`/`NumberColumn`/`IconColumn`/`ImageColumn`/`RelationshipColumn`/`ComputedColumn`) com fluent API e `Column::make($name)` factory (TABLE-003)
- `Arqel\Table\Filter` abstract + 6 tipos concretos (`SelectFilter`/`MultiSelectFilter`/`DateRangeFilter`/`TextFilter`/`TernaryFilter`/`ScopeFilter`) com `applyToQuery()` oracle e `apply(Closure)` override (TABLE-004)
- `Arqel\Table\TableQueryBuilder` orquestra search/filter/sort/eager-load/paginate da request → Eloquent query. Sort whitelisted contra colunas sortable; per_page validado contra getPerPageOptions; eager loading inferido de RelationshipColumn (TABLE-005)
- Esqueleto do pacote `arqel/form` (PHP 8.3+, Laravel 12|13, depende de `arqel/core`/`arqel/fields` via path repositories): `FormServiceProvider`, PSR-4 `Arqel\Form\` → `src/`, Pest + Orchestra Testbench, smoke tests verificando boot do provider e autoload do namespace (FORM-001)
- `Arqel\Form\Form` fluent builder (final): `make()`, `schema(array)`, `columns(int)` (clamp ≥ 1), `model(class-string<Model>)`, `inline(bool)`, `disabled(bool)`. `getFields()` flatten recursivo descendo em layout components; `toArray()` serializa schema com `kind: field|layout` para o payload Inertia (FORM-002)
- `Arqel\Form\Layout\Component` abstract base para layout components (Section/Fieldset/Grid/Columns/Group/Tabs/Tab): estado partilhado (`$schema`/`$columnSpan`/`$visibleIf`/`$canSee`), fluent `schema`/`columnSpan`/`visibleIf`/`canSee`, oracle `isVisibleFor(?Model)` que avalia `canSee` antes de `visibleIf`, template method `getTypeSpecificProps()` e `toArray()` final emitindo `{type, component, columnSpan, props}`. `Section` (heading/description/icon/collapsible/collapsed/columns/compact/aside; `collapsed()` implica `collapsible()`). `Fieldset` (legend + columns) (FORM-003, FORM-004 parcial)
- `Arqel\Form\Layout\Grid` (colunas fixas `int` ou breakpoint map `array<string,int>`, gap), `Arqel\Form\Layout\Columns` (atalho semântico para 2 colunas), `Arqel\Form\Layout\Group` (sem chrome visual; `orientation('horizontal'|'vertical')` com fallback para vertical em valores inválidos; `visibleIf`/`canSee` aplicam-se ao bloco) (FORM-004)
- `Arqel\Form\Layout\Tabs` (`tabs(array<Tab>)`, `defaultTab(id)` com fallback para o primeiro Tab, `vertical()`/`horizontal()` orientation) e `Arqel\Form\Layout\Tab` (`id`, `label`, `icon`, `badge(int|Closure)` resolvido em serialize-time com retornos não-int descartados) (FORM-005)
- SKILL.md do pacote `arqel/form` reescrito de raiz: status real (FORM-001..005 entregue, FORM-006/008 adiados por dependência em CORE-006), Key Contracts (Form fluent API + tabela dos 7 layout components), 3 exemplos copy-pasteáveis (Section+Grid, Tabs+visibility, Group invisible), Conventions e 5 Anti-patterns (FORM-010 parcial)
- Planejamento completo em `PLANNING/` (13 documentos, 328 tickets)
- Convenções operacionais em `CLAUDE.md` e `AGENTS.md`
