## Contexto

A cadeia v0.9.1 → v0.10.0 corrigiu 8 BUG-VAL bloqueantes em 5 sprints de hotfix. O padrão observado: **cada camada (render → URL → placeholder → form schema → panel prefix) falhou no primeiro contacto externo**. Os testes feature do monorepo (`apps/demo` Pest, `packages/*/tests`) **passam** em todas as camadas mas **não exercitam a UI**. Os Vitest unit tests cobrem só `ArqelIndexPage`. Sem cobertura E2E, novos caminhos não-testados continuam a acumular.

A sprint v0.10.0 fechou Fase 1 do roadmap — todos os bloqueantes resolvidos. v0.10.1 tem um único objectivo: **adicionar cobertura E2E que previna próximas cadeias de bugs latentes**, antes de abrir features novas Fase 2.

## Objectivos

1. Adicionar Playwright em `apps/demo` (chromium-only)
2. Implementar **smoke set de 10 cenários** cobrindo exactamente os caminhos onde a cadeia v0.9.x descobriu bugs
3. CI job novo **`Tests E2E (Playwright)`** em todo PR
4. Job aceite como **required check** em main branch protection
5. Publicar v0.10.1 (patch — só adiciona infra de testing)
6. **Fase 2 features pode arrancar** após v0.10.1 ship

## Não-objectivos (YAGNI)

- **Não cross-browser** (Firefox/Webkit) — chromium-only nesta sprint
- **Não visual regression** (snapshots de pixels) — só comportamento funcional
- **Não cobrir Cmd+K, Theme toggle, Categories** — sprint dedicada se houver demanda
- **Não Lighthouse/perf benchmarks**
- **Não fix BUG-VAL-005, 006, 008, 009, 013** — backlog
- **Não criar `apps/playground`** — `apps/demo` cumpre a função (PostResource + CategoryResource já configurados)
- **Não cobertura paralela ao monorepo** (separate workspace `packages-js/e2e`) — testes vivem com o app que testam

## Constraint operacional: economia de recursos

Política session-level mantida:

- Não rodar `pnpm test` global (Vitest)
- Não rebuilds de monorepo desnecessários
- Subagent dispatches consolidados (Fase 0+1+2 podem cair no mesmo subagent; release separado)
- Reusa `arqel-test` sandbox sem alterações

## Fase 0 — Setup Playwright em `apps/demo`

### Ficheiros tocados

- **Modify:** `apps/demo/package.json` — adicionar `@playwright/test`, scripts `test:e2e` + `test:e2e:install`
- **Create:** `apps/demo/playwright.config.ts`
- **Create:** `apps/demo/tests/e2e/setup.ts` — helper `resetDatabase()` usando `execFileSync` (sem shell)
- **Create:** `apps/demo/tests/e2e/fixtures.ts` — custom test fixture com `loggedInPage`
- **Modify:** `apps/demo/.gitignore` — adicionar `test-results/`, `playwright-report/`, `playwright/.cache/`
- **Modify:** `package.json` (root) — script `test:e2e` aliased para `pnpm --filter @arqel-dev/demo test:e2e`

### Decisões técnicas

- **Browser:** Chromium-only (mais rápido, mais previsível em CI)
- **Base URL:** `http://localhost:8000` (matching `php artisan serve`)
- **`webServer` automático:** Playwright arranca `php artisan serve --port=8000` se a porta livre, espera 200 em `/admin/login`
- **`workers: 1`** inicialmente (paralelismo introduz race com SQLite + reset DB; optimizar mais tarde se preciso)
- **Reset DB per-test:** `beforeEach` chama `resetDatabase()` que faz `php artisan migrate:fresh --seed` + `arqel:make-user` via **`execFileSync`** (sem shell, sem injection surface). ~2-3s por teste.
- **Login fixture:** custom test fixture `loggedInPage` faz login automático ao entrar no test, retorna `Page` autenticada. 9/10 specs usam-na (excepto `01-login` que testa o login em si).
- **Retries:** 1 em CI, 0 local. `trace: 'on-first-retry'` para debug.
- **Reporters:** `[['github'], ['html', { open: 'never' }]]` em CI; `'list'` local.

### `playwright.config.ts`

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests/e2e',
  fullyParallel: false,
  workers: 1,
  retries: process.env.CI ? 1 : 0,
  reporter: process.env.CI ? [['github'], ['html', { open: 'never' }]] : 'list',
  use: {
    baseURL: 'http://localhost:8000',
    trace: 'on-first-retry',
    headless: true,
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
  ],
  webServer: {
    command: 'php artisan serve --port=8000',
    url: 'http://localhost:8000/admin/login',
    reuseExistingServer: !process.env.CI,
    timeout: 30_000,
  },
});
```

### `setup.ts`

Uses `execFileSync` (no shell) for safety — commands hardcoded, no user input, but pattern follows codebase guidance.

```typescript
import { execFileSync } from 'node:child_process';
import { join } from 'node:path';

const APP_ROOT = join(__dirname, '..', '..');

/** Run an artisan command synchronously without spawning a shell. */
function runArtisan(args: string[]): void {
  execFileSync('php', ['artisan', ...args], {
    cwd: APP_ROOT,
    stdio: 'pipe',
  });
}

export function resetDatabase(): void {
  runArtisan(['migrate:fresh', '--seed', '--force']);
  runArtisan([
    'arqel:make-user',
    '--email=admin@arqel.test',
    '--password=password',
    '--name=Admin',
  ]);
}
```

### `fixtures.ts`

```typescript
import { test as base, expect, type Page } from '@playwright/test';
import { resetDatabase } from './setup';

interface Fixtures {

}

export const test = base.extend<Fixtures>({

});

export { expect };
```

### Padrão por spec (exemplo: pagination)

```typescript
import { test, expect } from './fixtures';

test.describe('BUG-VAL-001: pagination preserves perPage', () => {
  test('changing perPage to 10 then clicking Next shows different records', async ({ loggedInPage }) => {
    await loggedInPage.goto('/admin/posts');

    await expect(loggedInPage.locator('table tbody tr').first()).toBeVisible();

    const perPageSelect = loggedInPage.locator('select').first();
    await perPageSelect.selectOption('10');
    await loggedInPage.waitForURL(/per_page=10/);

    const firstTitleBefore = await loggedInPage.locator('table tbody tr').first().textContent();

    await loggedInPage.getByRole('button', { name: /next page/i }).click();
    await loggedInPage.waitForURL(/page=2/);

    const firstTitleAfter = await loggedInPage.locator('table tbody tr').first().textContent();
    expect(firstTitleAfter).not.toBe(firstTitleBefore);

    expect(loggedInPage.url()).toMatch(/per_page=10/);
  });
});
```

import { test, expect } from './fixtures';

test.describe('BUG-VAL-002 + 011 + 012: row Edit full lifecycle', () => {
  test('click Edit, see form inputs, save, persist', async ({ loggedInPage }) => {
    await loggedInPage.goto('/admin/posts');

    const editButton = loggedInPage.locator('table tbody tr').first()
      .getByRole('button', { name: /edit/i }).first();
    await editButton.click();

    // BUG-VAL-011: form must have actual input fields

    await expect(loggedInPage.locator('input[name="slug"]')).toBeVisible();

    await loggedInPage.waitForLoadState('networkidle');

    await loggedInPage.reload();
    await expect(loggedInPage.locator('input[name="title"]')).toHaveValue(newTitle);
  });
});

### Job `e2e` em `ci.yml`

```yaml
e2e:
  name: Tests E2E (Playwright)
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v6

    - name: Setup PHP
      uses: shivammathur/setup-php@v2
      with:
        php-version: '8.3'
        extensions: sqlite, mbstring, xml, curl
        coverage: none

    - uses: pnpm/action-setup@v6

    - name: Setup Node.js
      uses: actions/setup-node@v6
      with:
        node-version: ${{ env.NODE_VERSION }}
        cache: pnpm

    - name: Install Composer deps in apps/demo
      run: composer install --working-dir=apps/demo --no-interaction --prefer-dist

    - name: Install npm deps
      run: pnpm install --frozen-lockfile

    - name: Build frontend (apps/demo)
      run: pnpm --filter @arqel-dev/demo build

    - name: Cache Playwright browsers
      uses: actions/cache@v4
      with:
        path: ~/.cache/ms-playwright
        key: playwright-${{ runner.os }}-${{ hashFiles('apps/demo/package.json') }}

    - name: Install Playwright browsers
      run: pnpm --filter @arqel-dev/demo exec playwright install --with-deps chromium

    - name: Setup .env + SQLite + key
      working-directory: apps/demo
      run: |
        cp .env.example .env || true
        php artisan key:generate
        touch database/database.sqlite

    - name: Run E2E tests
      working-directory: apps/demo
      run: pnpm exec playwright test

    - name: Upload Playwright report on failure
      if: failure()
      uses: actions/upload-artifact@v4
      with:
        name: playwright-report
        path: apps/demo/playwright-report/
        retention-days: 14
```

### Branch protection

Após PR merge, adicionar `Tests E2E (Playwright)` aos required checks. Preferência: via GitHub UI (Settings → Branches → main → Edit protection rule → Required status checks → Search "Tests E2E"). Alternativa CLI requer JSON payload completo (mais arriscado para fix in-flight).

## Riscos e mitigações

| Risco | Mitigação |
|-------|-----------|
| Flake em selectors (timing race) | Fixtures usam `waitForURL` + `expect.toBeVisible({ timeout })`; `retries: 1` em CI |
| `php artisan migrate:fresh --seed` lento (~3s × 10 = 30s) | Aceitável; se tornar dor, mover para `globalSetup` per-file |
| Playwright install cache miss | `actions/cache` key inclui `apps/demo/package.json` |
| `webServer` config conflita com manual `php artisan serve` em dev | `reuseExistingServer: !process.env.CI` |
| Port 8000 já em uso em CI runner | Improvável; config tem timeout 30s |
| Required check bloqueia PRs se um teste flaky | Retries=1 + monitoring; se flake aparecer, fix imediato (não desactivar) |
| Selectors quebram quando UI muda | Cobertura é P0 path; mudanças de UI passam por PR review |
| `apps/demo` precisa ter PostResource + admin user | Já tem; `setup.ts` confirma via `arqel:make-user` |
| `execFileSync` em test setup — security concern | Hardcoded args, no user input, no shell expansion |
