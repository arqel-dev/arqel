# v0.9.3 actions URL resolution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stock actions (`Actions::edit/delete/view/restore/deleteBulk`) emit serialised payload com URLs e methods correctos quando usadas dentro de um Resource, e adicionar rota `POST admin/{resource}/bulk/{action}` para executar bulk actions sobre records selecionados — desbloqueando BUG-VAL-002 e BUG-VAL-003 fim-a-fim.

**Architecture:** Modificações PHP-only. `Action::toArray()` recebe opcionalmente um Resource (duck-typed para evitar ciclo `actions ↔ core`); quando o action está em estado stock (sem `url` nem `action(Closure)`), resolveStockUrl() emite URL+method baseados em convenção (verbo do action × type row/bulk). `InertiaDataBuilder::serializeMany()` propaga o resource. Nova rota bulk em `packages/core/routes/arqel.php` aponta para `ResourceController::bulkAction` que executa o callback do BulkAction (com chunking via `BulkAction::execute`) ou aplica fallback `whereIn->delete()` quando o action é o stock `deleteBulk`.

**Tech Stack:** PHP 8.3+, Laravel 12+, Pest 3, Composer 2, Packagist, NPM.

**Spec:** [`docs/superpowers/specs/2026-05-07-v0.9.3-actions-url-resolution-design.md`](../specs/2026-05-07-v0.9.3-actions-url-resolution-design.md)

**Constraint operacional (resource economy):** scope-limited Pest commands (`vendor/bin/pest <path>`), nunca `pest` global, nunca `pnpm test` global, subagent dispatches consolidados.

   /**
    * Serialise the action for transport to the React side.
    *
    * `$resource` (optional, duck-typed to keep arqel-dev/actions
    * decoupled from arqel-dev/core) lets the action emit the
    * conventional URL+method when no explicit `->url(...)` was
    * declared and the action name matches a stock verb (view,
    * edit, delete, restore for row; delete for bulk).
    *
    * @return array<string, mixed>
    */
   public function toArray(
       ?Authenticatable $user = null,
       mixed $record = null,
       ?object $resource = null,
   ): array {
       $url = $this->resolveUrl($record);
       $method = $this->method;

       if ($url === null && $resource !== null) {
           $stock = $this->resolveStockUrl($resource, $record);
           if ($stock !== null) {
               [$url, $method] = $stock;
           }
       }

       return array_filter([
           'name' => $this->name,
           'type' => $this->type,
           'label' => $this->label,
           'icon' => $this->icon,
           'color' => $this->color,
           'variant' => $this->variant,
           'method' => $method,
           'url' => $url,
           'tooltip' => $this->tooltip,
           'disabled' => $this->isDisabledFor($record) ?: null,
           'requiresConfirmation' => $this->isRequiringConfirmation() ?: null,
           'confirmation' => $this->getConfirmationConfig(),
           'form' => $this->hasForm() ? $this->getFormSchemaArray() : null,
           'modalSize' => $this->hasForm() ? $this->getModalSize() : null,
           'successNotification' => $this->successNotification,
           'failureNotification' => $this->failureNotification,
       ], fn ($v) => $v !== null);
   }

   /**
    * Stock convention URLs: when an action has no explicit url
    * but its name matches a known resource verb, generate the
    * URL by convention. Returns `[url, method]` or `null`.
    *
    * Resource is duck-typed via `::$slug` static property so
    * arqel-dev/actions does not depend on arqel-dev/core.
    *
    * @return array{0: string, 1: string}|null
    */
   private function resolveStockUrl(object $resource, mixed $record): ?array
   {
       $slug = $resource::$slug ?? null;
       if (! is_string($slug) || $slug === '') {
           return null;
       }

       $id = is_object($record) && method_exists($record, 'getKey')
           ? $record->getKey()
           : null;

       return match ([$this->type, $this->name]) {
           ['row', 'view']    => $id !== null ? ["/admin/{$slug}/{$id}", 'GET'] : null,
           ['row', 'edit']    => $id !== null ? ["/admin/{$slug}/{$id}/edit", 'GET'] : null,
           ['row', 'delete']  => $id !== null ? ["/admin/{$slug}/{$id}", 'DELETE'] : null,
           ['row', 'restore'] => $id !== null ? ["/admin/{$slug}/{$id}/restore", 'POST'] : null,
           ['bulk', 'delete'] => ["/admin/{$slug}/bulk/delete", 'POST'],
           default            => null,
       };
   }
   ```

/**
 * Serialise an array of duck-typed objects with `toArray()`.
 *
 * `$resource` (optional) is forwarded as the third argument to
 * `toArray($user, null, $resource)` so stock Actions can emit
 * resource-aware URLs (see arqel-dev/actions Action::toArray).
 *
 * @param array<int, mixed> $items
 * @return array<int, array<string, mixed>>
 */
private function serializeMany(
    array $items,
    ?Authenticatable $user = null,
    ?Resource $resource = null,
): array {
    $serialized = [];
    foreach ($items as $item) {
        if (! is_object($item) || ! method_exists($item, 'toArray')) {
            continue;
        }

        // Try the 3-arg signature first (Action.php in v0.9.3+);
        // gracefully fall back to 2-arg for any duck-typed object
        // that hasn't been updated.
        try {
            $serialized[] = $item->toArray($user, null, $resource);
        } catch (\ArgumentCountError) {
            $serialized[] = $item->toArray($user);
        }
    }
    return $serialized;
}
```

(If the existing body has different filtering logic, adapt — keep the existing semantics, only add the `$resource` propagation. The try/catch fallback is defensive in case a non-Action object also gets passed via `serializeMany`.)

If existing signature uses different param names or additional params, match those exactly. The key change is adding `?Resource $resource = null` and forwarding it to `toArray`.

```php
/**
 * Execute a bulk action over the selected records.
 *
 * Records are resolved from `record_ids` request input and
 * dispatched through the BulkAction's callback (when one was
 * declared via `->action(Closure)`), with chunking handled by
 * `BulkAction::execute()`.
 *
 * Stock fallback: when the action has no callback and is named
 * `delete`, perform a hard delete via `whereIn->delete()`.
 *
 * 4xx responses:
 *   - 404 if resource slug or action name not found
 *   - 403 if policy denies (via authorize())
 *   - back() with error flash when no records selected or
 *     the action has no callback and no stock fallback applies
 */
public function bulkAction(
    \Illuminate\Http\Request $request,
    string $resource,
    string $action,
): \Illuminate\Http\RedirectResponse {
    $instance = $this->resolveOrFail($resource);

    $table = method_exists($instance, 'table') ? $instance->table() : null;
    if ($table === null || ! method_exists($table, 'getBulkActions')) {
        abort(404);
    }

    $bulkActions = $table->getBulkActions();
    $bulkAction = null;
    foreach ($bulkActions as $candidate) {
        if (is_object($candidate)
            && method_exists($candidate, 'getName')
            && $candidate->getName() === $action) {
            $bulkAction = $candidate;
            break;
        }
    }

    if ($bulkAction === null) {
        abort(404);
    }

    /** @var class-string<\Illuminate\Database\Eloquent\Model> $modelClass */
    $modelClass = $instance::getModel();

    // Authorize — same pattern as destroy(). For bulk we authorise
    // against the model class (not a single record) because policies
    // typically gate by user role, not per-record state.
    $this->authorize('delete', $modelClass);

    $recordIds = (array) $request->input('record_ids', []);
    if ($recordIds === []) {
        return back()->with('error', 'No records selected.');
    }

    $records = $modelClass::whereIn('id', $recordIds)->get();

    $hasCallback = method_exists($bulkAction, 'hasCallback')
        && $bulkAction->hasCallback();

    if ($hasCallback && method_exists($bulkAction, 'execute')) {
        $bulkAction->execute($records, (array) $request->input('payload', []));
    } elseif ($action === 'delete') {
        // Stock Actions::deleteBulk() has no callback; do a hard delete
        // mirroring the row destroy() semantics.
        $modelClass::whereIn('id', $recordIds)->delete();
    } else {
        return back()->with('error', "Bulk action '{$action}' has no callback.");
    }

    $label = method_exists($bulkAction, 'getLabel') ? $bulkAction->getLabel() : $action;

    return back()->with('success', "{$label} executed on " . count($records) . ' records.');
}
```

(If `getModel()` does not exist on `Resource` base, use `$instance::$model` directly. Confirm during implementation by reading `Resource.php`.)
