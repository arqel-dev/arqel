## Contexto

`packages/tenant/` (PHP) já é production-ready: 29 PHP files, 157 Pest tests passing, version 0.11.0, integrado em CI release matrix. SKILL.md detalhado documenta todas as primitivas:

- `TenantManager` singleton + 5 resolvers concretos (Subdomain, Path, Header, Session, AuthUser)
- `BelongsToTenant` Eloquent trait + `TenantScope` global scope + `ScopedUnique` rule
- 3 scaffolders Artisan (registration, profile, billing)
- `TenantSwitcherController` com routes `POST /admin/tenants/{id}/switch` + `GET /admin/tenants/available`
- Adapters opt-in para `stancl/tenancy` e `spatie/laravel-multitenancy`
- Theming (CSS vars) + feature gates middleware

**Não está no metapackage** `arqel-dev/framework` — decisão deliberada, tenancy é opt-in (matched no PLANNING). **Sem JS surface concreta**: `<TenantSwitcher>` slot reservado em `Topbar.tsx` (`tenantSwitcher?: ReactNode`) mas sem implementação. **Sem demo end-to-end**.

Sprint v0.12.0 completa a integração frontend + reference app, mirror exacto do padrão WIDGETS v0.11.0.

## Objectivos

1. Implementar `<TenantSwitcher>` React component em `@arqel-dev/ui/shell`
2. Adicionar `TenantSummary` types em `@arqel-dev/types`
3. Criar `apps/tenant-demo/` reference app — Laravel skeleton minimal com PathResolver, single Resource (`ProjectResource`), tenant switcher integrado no Topbar
4. Adicionar 1 E2E spec (`01-tenant-switching.spec.ts`) cobrindo o caminho crítico (login → list scoped → switch → list scoped diferente)
5. Estender CI E2E job para correr ambos `apps/demo` + `apps/tenant-demo`
6. Publicar v0.12.0 (minor — `<TenantSwitcher>` em UI public API)
7. Re-validar in-place no `arqel-test` sandbox (mesmo que não use tenancy — verifica que upgrade não quebra)

## Não-objectivos (YAGNI)

- **Não re-implementar PHP** — 157 Pest tests verdes
- **Não adicionar `arqel-dev/tenant` ao metapackage** — fica opt-in (`composer require arqel-dev/tenant` separadamente). Apps consumers single-tenant não pagam o custo.
- **Não Subdomain/Header/Session/AuthUser resolvers** como exemplos no demo — PathResolver only para minimizar setup local
- **Não billing scaffolder demo**, não white-label theming demo, não feature gates demo
- **Não cross-browser E2E** (chromium-only)
- **Não Vitest unit tests para TenantSwitcher** (E2E cobre happy path; pattern v0.11.0)
- **Não fix BUG-VAL-005, 008, 009, 013** (backlog)
- **Não 3 specs E2E separados** (1 consolidado mantém cadência "1 E2E novo por feature" estabelecida em v0.10.1)

## Constraint operacional

Política session-level mantém-se:
- Subagent dispatches consolidados onde fizer sentido
- Reusa `arqel-test` sandbox via in-place upgrade (sandbox não muda para multi-tenant)
- Pause-on-tag policy ("C")
- Sem `pnpm test` global; scope-limited per package

Data flow:

```
[apps/tenant-demo/bootstrap/app.php]
  ├── withRouting(prefix: '/admin/{tenant}', ...)
  └── middleware('arqel.tenant') registered globally on /admin/{tenant}/*
        ↓
[ResolveTenantMiddleware via PathResolver]
  └── reads {tenant} segment → loads Tenant model → TenantManager::set()
        ↓
[Inertia request — HandleArqelInertiaRequests::share]
  └── shares { tenant: { current: TenantSummary, available: TenantSummary[] } }
        ↓
[Topbar receives tenantSwitcher prop]
  └── <TenantSwitcher current={...} available={[...]} />
        ↓
[Click on switcher item]
  └── router.post('/admin/tenants/{id}/switch')
        ↓ TenantSwitcherController dispatches TenantSwitched event + redirect
  └── client navigates to /admin/{newSlug}/{currentResource}
```

  ```typescript
  import type { TenantSummary } from '@arqel-dev/types/tenant';
  import { router } from '@inertiajs/react';
  import { type JSX } from 'react';
  import {
    DropdownMenu,
    DropdownMenuContent,
    DropdownMenuItem,
    DropdownMenuTrigger,
  } from '../shadcn/ui/dropdown-menu.js';
  import { Button } from '../action/Button.js';

  export interface TenantSwitcherProps {
    current: TenantSummary | null;
    available: ReadonlyArray<TenantSummary>;
    /** Path template for switching; defaults to `/admin/tenants/{id}/switch`. */
    switchUrl?: (tenantId: string | number) => string;
  }

  export function TenantSwitcher({
    current,
    available,
    switchUrl,
  }: TenantSwitcherProps): JSX.Element | null {
    if (!current || available.length <= 1) return null;

    const buildUrl = switchUrl ?? ((id) => `/admin/tenants/${id}/switch`);

    return (
      <DropdownMenu>
        <DropdownMenuTrigger asChild>
          <Button variant="ghost" size="sm">
            {current.label}
          </Button>
        </DropdownMenuTrigger>
        <DropdownMenuContent align="end">
          {available.map((t) => (
            <DropdownMenuItem
              key={t.id}
              onSelect={() => {
                router.post(buildUrl(t.id), {}, { preserveScroll: true });
              }}
            >
              {t.label}
              {t.id === current.id ? ' ✓' : ''}
            </DropdownMenuItem>
          ))}
        </DropdownMenuContent>
      </DropdownMenu>
    );
  }
  ```

### `HandleArqelInertiaRequests::share()`

```php
public function share(Request $request): array
{
    return array_merge(parent::share($request), [
        'tenant' => $this->tenantContext($request),
    ]);
}

private function tenantContext(Request $request): array
{
    $manager = app(\Arqel\Tenant\TenantManager::class);
    $current = $manager->current();
    $user = $request->user();

    return [
        'current' => $current ? ['id' => $current->id, 'label' => $current->name, 'slug' => $current->slug] : null,
        'available' => $user ? $user->tenants()->get()->map(fn ($t) => ['id' => $t->id, 'label' => $t->name, 'slug' => $t->slug])->toArray() : [],
    ];
}
```

## Fase 4 — E2E spec + CI matrix

### `apps/tenant-demo/tests/e2e/01-tenant-switching.spec.ts`

```typescript
import { test, expect } from './fixtures';

test.describe('Tenant switching (PathResolver)', () => {
  test('login → list scoped to tenant A → switch → list scoped to tenant B', async ({ loggedInPage }) => {
    // After login, fixture lands on /admin/acme/projects
    await expect(loggedInPage).toHaveURL(/\/admin\/acme\/projects/);
    await expect(loggedInPage.locator('table tbody tr')).toHaveCount(5);

    // Capture project names from Acme
    const acmeNames = await loggedInPage.locator('table tbody tr td:first-child').allInnerTexts();
    expect(acmeNames).toHaveLength(5);

    // Click tenant switcher (Topbar dropdown)
    await loggedInPage.getByRole('button', { name: /acme/i }).click();
    await loggedInPage.getByRole('menuitem', { name: /globex/i }).click();

    // URL switched
    await expect(loggedInPage).toHaveURL(/\/admin\/globex\/projects/);

    // List now shows different rows
    await expect(loggedInPage.locator('table tbody tr')).toHaveCount(5);
    const globexNames = await loggedInPage.locator('table tbody tr td:first-child').allInnerTexts();
    expect(globexNames).toHaveLength(5);

    // Acme and Globex projects must be different (factory generates unique names)
    const overlap = acmeNames.filter((n) => globexNames.includes(n));
    expect(overlap).toHaveLength(0);
  });
});
```
