**v0.10.0 é o "quality foundation release":** elimina o último débito infra-estrutura (release pipeline) + desbloqueia o caminho create/edit forms. Tag minor (não patch) por causa do scope: novo campo público `ResourceMeta.panelPath` + mudança de comportamento no `Form::toArray`.

## Fase 0 — BUG-VAL-007: Release pipeline framework subtree

### Causa raiz

`.github/workflows/release.yml` matrix usa **string list** para o nome dos pacotes:

```yaml
matrix:
  package:
    - actions
    - ai
    # ... 19 entries
```

Steps usam `${{ matrix.package }}` literal e `--prefix="packages/$PKG"`. O meta-package `framework` está em `packages/arqel/` (não `packages/framework/`), portanto não pode ser adicionado à lista plana — `--prefix=packages/framework` não existe.

### Fix

Converter matrix para `include` com object entries:

```yaml
strategy:
  fail-fast: false
  matrix:
    include:
      - { name: actions, path: packages/actions }
      - { name: ai, path: packages/ai }
      - { name: audit, path: packages/audit }
      - { name: auth, path: packages/auth }
      - { name: cli, path: packages/cli }
      - { name: core, path: packages/core }
      - { name: export, path: packages/export }
      - { name: fields, path: packages/fields }
      - { name: fields-advanced, path: packages/fields-advanced }
      - { name: form, path: packages/form }
      - { name: marketplace, path: packages/marketplace }
      - { name: mcp, path: packages/mcp }
      - { name: nav, path: packages/nav }
      - { name: realtime, path: packages/realtime }
      - { name: table, path: packages/table }
      - { name: tenant, path: packages/tenant }
      - { name: versioning, path: packages/versioning }
      - { name: widgets, path: packages/widgets }
      - { name: workflow, path: packages/workflow }
      - { name: framework, path: packages/arqel }   # NEW
```

Steps actualizados:

| Step | Old | New |
|------|-----|-----|
| `Ensure sub-repo exists` | `gh repo view "$GH_ORG/$PKG"` | `gh repo view "$GH_ORG/$NAME"` |
| `Compute split SHA` | `splitsh-lite --prefix="packages/$PKG"` | `splitsh-lite --prefix="$PATH"` |
| `Push split to sub-repo` | `https://...github.com/$GH_ORG/$PKG.git` | `https://...github.com/$GH_ORG/$NAME.git` |
| All env: | `PKG: ${{ matrix.package }}` | `NAME: ${{ matrix.name }}\nPATH: ${{ matrix.path }}` |
| Job name (visible in UI) | `Split PHP — ${{ matrix.package }}` | `Split PHP — ${{ matrix.name }}` |

(`PATH` é um nome reservado no shell — usar `PKG_PATH` ou similar para evitar mascarar `$PATH`.)

### Validação

A própria release v0.10.0 (Fase 3) é o teste deste fix. Se o release workflow correr com sucesso e o job `Split PHP — framework` aparecer com `succeed`, BUG-VAL-007 confirmed fixed. Se falhar, fazer manual split como nas v0.9.x e debugar.

### Critério de saída Fase 0

`release.yml` actualizado, PR aberto, CI passa (CI matrix de testes não usa este workflow — só dispara em tag push, mas typecheck/lint do YAML deve passar).

## Fase 1 — BUG-VAL-011: Form schema recursive serialization

### Causa raiz

`packages/form/src/Form.php:155-179` actual:

```php
public function toArray(): array
{
    $schema = [];
    foreach ($this->schema as $item) {
        if ($item instanceof Field) {
            $schema[] = ['kind' => 'field', 'name' => $item->getName(), 'type' => $item->getType()];
        } elseif ($item instanceof Component) {
            $schema[] = ['kind' => 'layout', ...$item->toArray()];
        }
    }
    return [
        'schema' => $schema,
        'columns' => $this->columns,
        'model' => $this->model,
        'inline' => $this->inline,
        'disabled' => $this->disabled,
    ];
}
```

Itera só top-level. Component spread (`...$item->toArray()`) emite type/component/columnSpan/props mas **não emite children**. `Component::toArray()` é deliberadamente "without descending the schema" (comentário linha 102 de `Component.php`).

Frontend types (`packages-js/types/src/forms.ts`) declaram:

```typescript
interface LayoutBase<TType, TProps> {
  kind: 'layout';
  type: TType;
  component: string;
  columnSpan: number | string;
  props: TProps;
  schema?: SchemaEntry[];  // recursive
}
```

`<FormRenderer>` walka esta `schema?` quando layout. Vazia → não renderiza children.

### Fix

Refactor mínimo de `Form::toArray()` extraindo helper recursivo:

```php
public function toArray(): array
{
    return [
        'schema' => $this->serializeSchema($this->schema),
        'columns' => $this->columns,
        'model' => $this->model,
        'inline' => $this->inline,
        'disabled' => $this->disabled,
    ];
}

/**
 * Recursively serialise a schema array (top-level or component children).
 * Components emit `entry.schema` populated with their child entries; Form
 * descends — Component::toArray itself remains "without descending the schema".
 *
 * @param array<int, mixed> $items
 * @return array<int, array<string, mixed>>
 */
private function serializeSchema(array $items): array
{
    $entries = [];
    foreach ($items as $item) {
        if ($item instanceof Field) {
            $entries[] = [
                'kind' => 'field',
                'name' => $item->getName(),
                'type' => $item->getType(),
            ];
        } elseif ($item instanceof Component) {
            $entries[] = [
                'kind' => 'layout',
                ...$item->toArray(),
                'schema' => $this->serializeSchema($item->getSchema()),
            ];
        }
    }
    return $entries;
}
```

`entry.schema` aparece **depois** do spread `...$item->toArray()` — qualquer `schema` que o Component eventualmente emita é sobrescrito intencionalmente. Defensivo: actualmente Component::toArray nunca emite `schema`, mas a ordem garante o comportamento desejado.

`Component::toArray()` permanece intocado. Contrato fica explícito: **Form descende, Component só emite as suas próprias props**.

### Tests

Create `packages/form/tests/Unit/FormSchemaRecursiveSerializationTest.php` com 3 testes:

1. **Section com Fields children** — `Form::toArray()['schema'][0]['schema']` tem 2 entries `kind=field`
2. **Nested layouts** — `Section > Fieldset > Field` desce 3 níveis correctamente
3. **Layout sem children** — emite `schema: []` (não omite a key)

`makeStubField($name, $type)` helper depende do pattern dos testes existentes. Plan inspecciona `packages/form/tests/` durante implementation step 1.

### Critério de saída Fase 1

3 testes Pest novos passam. Suite existente em `packages/form` continua verde scope-limited.

## Fase 2 — BUG-VAL-012: Panel prefix em Edit/Create pages via ResourceMeta

### Causa raiz

`packages-js/ui/src/pages/ArqelEditPage.tsx:46`:

```typescript
router.put(`/${slug}/${id}`, ...);
```

URL relativa, sem prefix `/admin/`. Resolve para `/posts/1` → 404.

`ArqelCreatePage.tsx:46` mesma issue:

```typescript
router.post(`/${slug}`, ...);
```

Panel path não é hardcoded — `Panel::path()` permite custom (default `'admin'`). `ArqelServiceProvider::registerResourceRoutes()` resolve via:

```php
$configPath = config('arqel.path', 'admin');
$path = $panel?->getPath() ?? (is_string($configPath) ? $configPath : 'admin');
```

### Fix

Server emite o panel path no `ResourceMeta` shape; frontend consome.

#### Step 1: PHP — `InertiaDataBuilder::resourceMeta()`

Adicionar resolução de panel path (espelhar lógica de `registerResourceRoutes`):

```php
private function resourceMeta(Resource $resource): array
{
    $panel = app(PanelRegistry::class)->getCurrent();
    $configPath = config('arqel.path', 'admin');
    $rawPath = $panel?->getPath() ?? (is_string($configPath) ? $configPath : 'admin');
    $panelPath = '/' . trim($rawPath, '/');

    return [
        'slug' => $resource::$slug,
        'label' => $resource::$label,
        'pluralLabel' => $resource::$pluralLabel,
        'navigationGroup' => $resource::$navigationGroup,
        'navigationIcon' => $resource::$navigationIcon ?? null,
        'panelPath' => $panelPath,  // NEW
        // ... outros campos existentes (match exact)
    ];
}
```

`/admin` formato: leading slash, no trailing — coerente com URL construction `${panelPath}/${slug}/${id}`.

#### Step 2: TS — `packages-js/types/src/resources.ts`

Adicionar `panelPath?: string` a `ResourceMeta`:

```typescript
export interface ResourceMeta {
  slug: string;
  label: string;
  pluralLabel: string;
  navigationGroup?: string | null;
  navigationIcon?: string | null;
  panelPath?: string;  // NEW (optional for backwards compat)
  // ... outros existentes
}
```

Optional para backwards compatibility — apps a correr v0.10.0 backend mas v0.9.x frontend (improvável mas possível) ainda funcionam com fallback.

#### Step 3: `ArqelEditPage.tsx:46`

```typescript
const submit = (e: FormEvent): void => {
  e.preventDefault();
  const id = String((record as { id?: string | number } | undefined)?.id ?? '');
  const slug = props.resource?.slug ?? '';
  const basePath = props.resource?.panelPath ?? '/admin';
  router.put(`${basePath}/${slug}/${id}`, form.data as Record<string, never>, {
    preserveScroll: true,
  });
};
```

#### Step 4: `ArqelCreatePage.tsx:46`

```typescript
const submit = (e: FormEvent): void => {
  e.preventDefault();
  const slug = props.resource?.slug ?? '';
  const basePath = props.resource?.panelPath ?? '/admin';
  router.post(`${basePath}/${slug}`, form.data as Record<string, never>, {
    preserveScroll: true,
  });

```

## Riscos e mitigações

| Risco | Mitigação |
|-------|-----------|
| Matrix `include` quebra outro job no `release.yml` | Inspeccionar dependentes da matrix `package` antes do PR; YAML lint local + CI catch |
| `PATH` env var conflict com shell built-in | Usar `PKG_PATH` ou similar; YAML literal `$PATH` em shell sem quotes faria mascarar |
| `Form::toArray` recursive quebra testes existentes que assertam shape antigo | Scope-limited Pest run no `packages/form` apanha imediato |
| `panelPath` ausente em payloads Inertia non-table (e.g. arqel:install initial bootstrap) | Frontend usa `?? '/admin'` fallback; backwards-compat |
| `PanelRegistry::getCurrent()` retorna null em testes que não bootstrap panel | Resolução cai em `config('arqel.path', 'admin')` default — coberto |
| Walkthrough descobre 5º bug no caminho create/edit | Decisão case-by-case: bloquear v0.10.0 ou roll para v0.10.1 |
| Release v0.10.0 falha (mesmo bug release.yml não corrigido) | Manual split como nas v0.9.x + investigar |
| Frontend changes em Edit/Create page sem teste — regressão silenciosa | Walkthrough manual obrigatório; v0.10.1 E2E captura definitivamente |
