# v0.10.1 Playwright E2E smoke set Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Playwright E2E smoke set in `apps/demo/tests/e2e/` covering 10 critical CRUD paths (login, list, filter, sort, search, pagination, row Edit/Delete, bulk Delete) and wire it as a required CI check `Tests E2E (Playwright)`, blocking PR merges that regress these paths.

**Architecture:** Playwright lives inside `apps/demo` as dev-dep, browser=chromium-only, single worker. Each test starts from `migrate:fresh --seed` via `execFileSync` (no shell, hardcoded args) and uses a custom `loggedInPage` fixture. CI job mirrors the existing Tests JS Vitest job structure but adds PHP setup, builds the demo, installs chromium with cached browsers, and runs `playwright test`. Branch protection main gets a 4th required check after the workflow lands.

**Tech Stack:** Playwright `^1.48.0`, TypeScript 5.9, Node 20.9+/22, PHP 8.3+, Laravel 13, SQLite, GitHub Actions, pnpm 10.

**Spec:** [`docs/superpowers/specs/2026-05-08-v0.10.1-playwright-e2e-design.md`](../specs/2026-05-08-v0.10.1-playwright-e2e-design.md)

**Constraint operacional (resource economy):** scope-limited Pest/Vitest runs (não touch global); subagent dispatches consolidados (Fase 0+1+2 num único subagent); reuse `arqel-test` sandbox without changes.

```typescript
import { execFileSync } from 'node:child_process';
import { join } from 'node:path';

const APP_ROOT = join(__dirname, '..', '..');

/**
 * Run an artisan command synchronously without spawning a shell.
 * All arguments are hardcoded — no user input flows through.
 */
function runArtisan(args: string[]): void {
  execFileSync('php', ['artisan', ...args], {
    cwd: APP_ROOT,
    stdio: 'pipe',
  });
}

/**
 * Reset the demo database to a known state for each E2E test.
 *
 * Runs `migrate:fresh --seed --force` then re-creates the admin
 * user (the seeder doesn't, since arqel:make-user is the canonical
 * path used by all sandbox upgrades in v0.9.x and v0.10.0).
 */
export function resetDatabase(): void {
  runArtisan(['migrate:fresh', '--seed', '--force']);
  runArtisan([
    'arqel:make-user',
    '--email=admin@arqel.test',
    '--password=password',
    '--name=Admin',
  ]);
}
```
